## 一、DNS 是什么——为什么要搞这一套

### 问题

```
你的代码跑在你自己电脑上（localhost:8000），只有你能访问。
你想让别人也能访问 → 需要把 localhost"挂到"公网上。
```

### 解决办法——三层结构

```
用户                     中间人                   你的电脑
trufflekit.com ───→ Cloudflare ───→ localhost:8000
   域名                 隧道                 项目代码
```

### 每一层负责什么

| 层 | 谁提供 | 干什么 | 花钱吗 |
|---|--------|--------|--------|
| **域名** | 阿里云 | 给用户一个好记的名字 | 85 元/年 |
| **DNS** | Cloudflare | 告诉全世界"trufflekit.com 归 Cloudflare 管" | 免费 |
| **Tunnel** | Cloudflare | 在你的电脑和 Cloudflare 之间开一条通道 | 免费 |
| **SSL** | Cloudflare | 自动 HTTPS，加密数据传输 | 免费 |
| **服务器** | 你的电脑 | 跑项目代码 | 你本来就付了电费 |

### 为什么不用云服务器

传统做法：
```
租一台阿里云 ECS（50 元/月）
在上面装 Python、装代码、跑起来
用户访问 → 服务器
```

你的做法：
```
用 Cloudflare Tunnel 把自家电脑连到公网
不需要租服务器
但电脑不能关机，关机了网站就打不开了
```

Tunnel 适合开发阶段、演示、小规模使用。
用户多了之后还是要换云服务器。

## 二、Cloudflare Tunnel 原理

### 普通连接 vs Tunnel

```
普通连接（被运营商拦截）：
用户 → 你的公网 IP (58.16.61.185) → 运营商拦截 ✗

Tunnel（绕过拦截）：
用户 → Cloudflare（国外服务器）→
      ──────────────────────────
      Cloudflare 主动连你（你电脑主动发起的连接）
        ↑ 运营商不拦截出站连接
      ──────────────────────────
      你的 cloudflared 进程 → localhost:8000 ✓
```

关键区别：**不是你等别人来连接，而是你主动去连接 Cloudflare。**

就像：
```
普通方式：你在家里等快递员送快递 → 快递员进不来（被拦）
Tunnel：你主动把东西送到快递站 → 用户去快递站拿
```

### cloudflared 是什么

cloudflared 是 Cloudflare 提供的一个小工具，你的电脑上运行的"隧道程序"。

```
cloudflared 启动后：
  1. 主动连上 Cloudflare 的服务器
  2. 保持这个连接不断开（长连接）
  3. 当有用户访问 trufflekit.com：
     Cloudflare 收到请求 → 通过这个长连接传给 cloudflared
     cloudflared 把请求转发到 localhost:8000
```

## 三、开发流程——你平时怎么工作

### 启动项目的标准流程

```bash
# 1. 启动后端
cd ~/Truffle/platform-backend
nohup uvicorn app.main:app --host 0.0.0.0 --port 8000 > /tmp/backend.log 2>&1 &

# 2. 构建前端（每次改了前端代码都要重新构建）
cd ~/Truffle/Nexus\ AI
npm run build

# 3. 检查后端是否在运行
curl http://localhost:8000/api/v1/health
# → 应该返回 {"status":"ok"}

# 4. 检查 ngrok 或 Cloudflare Tunnel 是否在运行
# 如果是 ngrok：访问 https://xxx.ngrok-free.dev
# 如果是 Cloudflare：访问 https://trufflekit.com
```

### 每次改完代码后

```bash
# 前端改了什么 → 重新构建
cd ~/Truffle/Nexus\ AI && npm run build
# dist/ 目录会更新，后端自动 serve 新的文件

# 后端改了代码 → 重启后端
kill $(lsof -ti:8000)  # 杀掉旧进程
nohup uvicorn app.main:app --host 0.0.0.0 --port 8000 > /tmp/backend.log 2>&1 &
```

### 重新构建前端时发生了什么

```bash
npm run build
# 1. TypeScript 编译器检查类型错误
# 2. Vite 把 TSX 文件编译成 JS
# 3. Vite 合并所有 JS 文件成一个（tree-shaking 去掉没用到的代码）
# 4. Vite 压缩 CSS
# 5. 输出到 dist/ 目录
#
# 产出：
dist/index.html             0.65 KB
dist/assets/index-xxx.js    255 KB   ← 所有 JS 打包成一个文件
dist/assets/index-xxx.css   23 KB    ← 所有 CSS 打包成一个文件
```

## 四、常见问题

### Q: 电脑关机了网站还能访问吗？

不能。你的代码跑在你的电脑上，关了机网站就访问不了。Tunnel 也会断开。

Cloudflare 会缓存一些静态页面，但动态内容（登录、提需求）会显示错误。

### Q: 重启电脑后怎么恢复？

```bash
# 1. 启动后端
cd ~/Truffle/platform-backend
nohup uvicorn app.main:app --host 0.0.0.0 --port 8000 > /tmp/backend.log 2>&1 &

# 2. 如果用的是 ngrok，重启隧道
nohup ngrok http 8000 --log=stdout > /tmp/ngrok.log 2>&1 &

# 3. 如果用的是 Cloudflare Tunnel
#    cloudflared 服务应该会自动启动（如果设了系统服务）
#    手动启动：
sudo cloudflared service install <token>  # 只需要装一次
sudo systemctl start cloudflared           # 重启后启动
```

### Q: 怎么知道网站挂了没？

```bash
# 在 WSL 里检查
curl http://localhost:8000/api/v1/health
# 返回 200 → 正常
# 连不上 → 后端挂了，重启

# 从外部检查
curl https://trufflekit.com/api/v1/health
# 返回 200 → 公网正常
# 连不上 → Tunnel 挂了
```

### Q: 跑测试

```bash
# 后端测试
cd ~/Truffle/platform-backend
python3 -m pytest tests/ -v

# 前端测试
cd ~/Truffle/Nexus\ AI
npm test
```

### Q: 换了网络（比如从家里到咖啡店）怎么办？

WSL 的 IP 会变，ngrok/Cloudflare Tunnel 会自动重连。

如果用了 Windows 端口转发（`netsh portproxy`），需要更新转发地址。

## 五、安全注意事项

| 不要做的事 | 为什么 |
|-----------|--------|
| **不要把 SECRET_KEY 传到 GitHub** | 别人能伪造管理员登录 |
| **不要用默认的 admin/admin123 密码** | 任何人都能登录 |
| **不要在微信/QQ 里分享 ngrok 地址** | ngrok 地址是公开的，看到的人都能访问 |
| **不要把 API Key 写在前端代码里** | 用户 F12 就能看到 |

## 六、如何调试

### 前端调试

```
浏览器 F12 → Console 标签：
  能看到 JS 错误、console.log 输出的信息

浏览器 F12 → Network 标签：
  能看到每个 API 请求的状态码、返回数据
  ↓
  某个请求返回 500 → 后端有 bug
  某个请求返回 401 → token 过期了
  某个请求挂了（pending）→ 后端没启动
```

### 后端调试

```bash
# 查看后端日志
tail -f /home/zhuzhu/Truffle/platform-backend/server.log

# 直接测试 API
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# 查看响应
# {"access_token":"eyJ...","role":"admin"}  → 正常
# {"detail":"Invalid credentials"}          → 密码错
# 连不上 → 后端没启动
```

### Tunnel 调试

```bash
# ngrok 状态检查
curl http://localhost:4040/api/tunnels
# → 能看到公网地址和状态

# Cloudflare Tunnel 日志
sudo journalctl -u cloudflared -n 50
```

## 学习要点总结

### 部署相关的核心概念

1. **域名**——给用户记住的名字，不是技术必须但用户体验必须
2. **DNS**——把域名指向你的服务器（或 Cloudflare）
3. **Tunnel**——绕过运营商封锁，不需要公网 IP 也不需要云服务器
4. **HTTPS**——Cloudflare 免费提供，自动加密
5. **生产构建**——`npm run build` 把代码打包成浏览器能高效加载的文件
6. **重启流程**——后端改了要重启，前端改了要构建
7. **调试工具**——浏览器 F12 看请求，tail -f 看日志，curl 测试 API
