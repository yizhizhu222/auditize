# 前后端连接——前端怎么和后端说话

## 一句话说清楚

```
前端（浏览器里的 React）→ 发 HTTP 请求 → 后端（服务器上的 FastAPI）
                  ↑                                    │
                  └── 拿到数据，更新页面 ←────── 处理完返回
```

前端和后端是两个完全独立的程序，通过 **HTTP 请求**通信。

---

## 通信方式——HTTP 请求

### 什么是 HTTP 请求？

HTTP = 前端和后端之间"写信"的格式。

```
前端写的信：
  POST /api/v1/auth/login HTTP/1.1
  Content-Type: application/json
  Authorization: Bearer eyJhbGciOiJIUzI1NiIs...

  {"username": "admin", "password": "admin123"}

后端回的信：
  HTTP/1.1 200 OK
  Content-Type: application/json

  {"access_token": "eyJhbGciOiJIUzI1NiIs...", "role": "admin"}
```

信封包括：
- **方法**（POST/GET/PUT/DELETE）——表示"我要干什么"
- **路径**（/api/v1/auth/login）——表示"我要找谁"
- **Headers**（Content-Type, Authorization）——附件说明
- **Body**（JSON 数据）——信的内容

### 状态码是什么意思

| 状态码 | 含义 | 你项目里常见的场景 |
|--------|------|------------------|
| **200** | 成功了 | 登录成功、数据返回成功 |
| **201** | 创建成功 | 提交需求成功、创建团队成功 |
| **400** | 请求格式不对 | 用户名太短、密码为空 |
| **401** | 未登录/登录过期 | token 过期、没传 token |
| **403** | 没有权限 | 不是 admin 想进 admin 页面 |
| **404** | 找不到 | API 地址写错了、数据不存在 |
| **409** | 冲突 | 重复提交、已经在一个团队里了 |
| **500** | 服务器内部错误 | 后端代码出 bug 了 |

### 前端在代码里怎么发请求

```tsx
// 用 fetch（浏览器原生 API，不需要额外装包）

// POST 请求——发数据给后端
const res = await fetch('/api/v1/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ username: 'admin', password: 'admin123' }),
})

// GET 请求——从后端拿数据
const res = await fetch('/api/v1/team/my', {
  headers: { Authorization: 'Bearer ' + token },
})

// 处理返回数据
const data = await res.json()  // 把 JSON 字符串转成 JavaScript 对象
if (res.ok) {
  // 成功了，更新页面
  setTeam(data)
} else {
  // 失败了，显示错误
  setError(data.detail)
}
```

## API 路径规则

```
/api/v1/team/requests
  │    │   │     └── 资源名（requests = 需求）
  │    │   └── 模块名（team = 团队管理）
  │    └── 版本号（v1 = 第一版）
  └── 固定前缀（所有 API 都以 /api 开头）
```

版本号的作用：如果以后把 API 改了，旧版本还能用。
```
/api/v1/team/requests  ← 旧版客户端用这个
/api/v2/team/requests  ← 新版用这个
```

## 认证流程——请求怎么带上身份

### 用户登录时的完整流程

```
前端                          后端
 │                             │
 │  POST /api/v1/auth/login    │
 │  {username, password}       │
 │ ──────────────────────────→ │ 验证密码
 │                             │
 │  200 {access_token, role}   │ 签发 JWT
 │ ←────────────────────────── │
 │                             │
 │  localStorage.setItem(      │ 前端把 token 存起来
 │    'nexus-auth-token',      │
 │    data.access_token        │
 │  )                          │
 │                             │
 │  GET /api/v1/team/my        │
 │  Authorization: Bearer xxx  │ 每次请求带上 token
 │ ──────────────────────────→ │ 验证 JWT
 │                             │
 │  200 {in_team: true, ...}   │ 返回数据
 │ ←────────────────────────── │
```

### 前端怎么管理认证状态

```tsx
// App.tsx——页面一加载就检查
function App() {
  const [isLoggedIn, setIsLoggedIn] = useState(false)

  useEffect(() => {
    const token = localStorage.getItem('nexus-auth-token')
    if (!token) {
      setIsLoggedIn(false)
      return
    }
    // 验证 token 是否过期
    fetch('/api/v1/auth/verify', {
      headers: { Authorization: `Bearer ${token}` }
    })
    .then(r => r.ok ? setIsLoggedIn(true) : setIsLoggedIn(false))
    .catch(() => setIsLoggedIn(false))
  }, [])

  // token 有效 → 显示主页面（Dashboard）
  // token 无效 → 显示登录页（Login）
  return isLoggedIn ? <Dashboard /> : <Login />
}
```

### localStorage 是什么

```javascript
// localStorage = 浏览器的"小本本"，关掉网页数据还在
localStorage.setItem('nexus-auth-token', 'eyJhbGciOi...')  // 存
localStorage.getItem('nexus-auth-token')                     // 取
localStorage.removeItem('nexus-auth-token')                  // 删
```

你的前端用 localStorage 存了：

| key | 存什么 |
|-----|--------|
| `nexus-auth-token` | JWT 令牌 |
| `nexus-auth-role` | 用户角色（user/admin） |
| `nexus-settings-openai` | OpenAI API Key |
| `nexus-theme-preset` | 用户选的主题 |
| `nexus-onboarding-done` | 新手引导完成状态 |

## API 代理——开发 vs 生产

### 开发时的代理

```tsx
// 你在开发时，前端跑在 localhost:5173，后端跑在 localhost:8000
// 前端发请求到 /api/v1/team/my
// 浏览器说："这是跨域请求！"

// 解决办法：在 vite.config.ts 配代理
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8000',  // 请求转发到这里
      changeOrigin: true,
    }
  }
}

// 现在前端请求 /api/v1/team/my
// Vite dev server 收到请求 → 转发到 localhost:8000
// 浏览器以为是同源请求 → 不报跨域错误
```

### 生产时的处理

```python
# 生产部署时，后端直接 serve 前端文件
# 不需要代理

if FRONTEND_DIST.exists():
    # 用户访问 / → 返回前端 index.html
    # 用户访问 /assets/xxx.js → 返回前端 JS 文件
    # 用户访问 /api/v1/xxx → 路由到后端 API 处理器

    # SPA 路由兜底：
    # 用户直接访问 /team → 后端找不到这个路径
    # 应该返回 index.html，让前端 JS 处理
    @app.exception_handler(404)
    async def spa_fallback(request, exc):
        if not request.url.path.startswith("/api/"):
            return FileResponse("index.html")
```

### CORS 在生产环境

```
生产环境不需要 CORS 代理，因为前端和后端在同域名下：
  https://trufflekit.com           → 前端
  https://trufflekit.com/api/v1/   → 后端（同域名、不同路径）
  → 浏览器不报跨域错误
```

但你的 CORS 配置还是有用的——其他域名（比如 ngrok）也能访问你的 API。

## 数据格式——JSON

### 什么是 JSON？

JSON = JavaScript Object Notation，前端和后端交换数据的"通用语言"。

```
后端返回的数据（JSON）：
{
  "in_team": true,
  "team_id": "abc-123",
  "name": "我的团队",
  "members": [
    {"user_id": 1, "username": "admin", "role": "owner"},
    {"user_id": 2, "username": "张三", "role": "member"}
  ]
}

前端拿到后直接当 JavaScript 对象用：
const team = await res.json()
console.log(team.name)          // "我的团队"
console.log(team.members[0].username)  // "admin"
```

### 为什么用 JSON 不是 XML？

| 对比 | JSON | XML |
|------|------|-----|
| 长度 | 短 | 长 |
| 可读性 | 好 | 差 |
| JS 解析 | 一行 json() | 要写解析器 |
| Python 解析 | 一行 json.loads() | 要写解析器 |

JSON 是目前最主流的 API 数据格式。

## 项目里的典型数据流

### 场景：用户提一个新需求

```
前端 TeamPage.tsx：
  1. 用户在表单里输入：title="订单管理", description="需要一个订单管理后台"
  2. 点击"提交"
  3. 代码：

     const res = await fetch('/api/v1/team/requests', {
       method: 'POST',
       headers: { Authorization: 'Bearer ' + token },
       body: JSON.stringify({ title: '订单管理', description: '需要一个订单管理后台' }),
     })
     const data = await res.json()
     // data.request_id = "uuid-xxx"  ← 后端返回的新需求 ID

  4. 刷新需求列表
     fetchRequests()  // 重新请求 GET /api/v1/team/requests

后端 team.py：
  1. 收到请求
  2. 验证 JWT → 知道是哪个用户
  3. 查数据库 → 知道用户属于哪个团队
  4. 验证数据 → title 不为空
  5. 生成 UUID、记录时间
  6. 写入数据库

     INSERT INTO feature_requests (id, team_id, user_id, title, status, created_at)
     VALUES ('uuid-xxx', 'team-abc', 1, '订单管理', 'pending', '2026-05-31...')

  7. 返回给前端
```

### 场景：生成代码后下载 ZIP

```
前端 Dashboard.tsx：
  1. 用户点击 "Download ZIP"
  2. 代码：

     const res = await fetch(`/api/v1/export/${taskId}`, {
       headers: { Authorization: 'Bearer ' + token },
     })
     const blob = await res.blob()
     // 创建一个虚拟链接，触发浏览器下载
     const url = URL.createObjectURL(blob)
     const a = document.createElement('a')
     a.href = url
     a.download = 'project.zip'
     a.click()

后端 export.py：
  1. 收到 GET /api/v1/export/{task_id}
  2. 查数据库 → 找到生成的代码
  3. 判断语言
  4. 生成项目文件（main.py + Dockerfile + README）
  5. 打包成 ZIP
  6. 返回 ZIP 文件（Content-Type: application/zip）
```

## 学习要点总结

### 前后端连接的核心知识

1. **HTTP 协议**——前端后端通信的"语言"，GET/POST/PUT/DELETE 四种方法
2. **RESTful API**——用路径表示资源，用方法表示操作
3. **JWT 认证**——登录后拿 token，每次请求带上 token
4. **JSON 数据格式**——前后端交换数据的"通用语言"
5. **CORS 策略**——浏览器限制跨域请求的安全机制
6. **API 代理**——开发时解决跨域问题的方法
7. **SPA 路由兜底**——确保用户直接输入 URL 也能正确显示页面
