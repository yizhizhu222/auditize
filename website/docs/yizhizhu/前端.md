# 前端——Nexus AI Platform

## 这个项目是做什么的？

一个面向小团队的**需求协作平台**。不懂代码的成员可以提交功能需求，技术成员可以审核、AI 生成代码，防止重复造轮子。

| 技术 | 用途 | 为什么选它 |
|------|------|-----------|
| **React 18** | UI 框架 | 最主流的前端框架，组件化开发，生态成熟 |

| 你没用的 | 原因 |
|---------|------|
| Next.js | 不需要服务端渲染，你的后端是 FastAPI |
| Redux | 项目规模不大，React Context 够用 |

| Axios | fetch 是浏览器原生 API，够用了 |

## 核心概念解释

### 什么是 SPA（单页应用）？

传统网站：点一个链接 → 浏览器重新加载整个页面。
SPA：点一个链接 → 只更新页面的一部分，不重新加载。

```
传统网站：
  用户点击 → 请求服务器 → 服务器返回整个 HTML → 浏览器重新渲染

SPA（你的项目）：
  用户点击 → JavaScript 接管 → 只更新变化的部分 → 不刷新页面
```

你的 `index.html` 只有这一个文件：
```html
<div id="root"></div>
<script src="/src/main.tsx"></script>
```
所有页面内容都是 JavaScript 动态生成的。

### 什么是 JSX？

JSX 就是**在 JavaScript 里写 HTML**。

```tsx
// 这是 JSX —— 在 JS 文件里直接写 <div>
const element = <div className="text-red">Hello</div>

// 底层会被编译成：
const element = React.createElement('div', { className: 'text-red' }, 'Hello')
```

方便之处：你不用分开写 HTML 文件和 JS 文件，一个组件文件里包含了样式、结构和逻辑。

### 什么是组件？

组件 = 一段可复用的 UI 代码。

```tsx
// 这是"一个组件"——可以单独使用，也可以嵌套在其他组件里
function LoginButton() {
  const [clicked, setClicked] = useState(false)
  return <button onClick={() => setClicked(true)}>登录</button>
}
```

你的项目有 11 个组件，按职责分为：

| 组件 | 职责 | 一句话说明 |
|------|------|-----------|
| `Login.tsx` | 登录注册 | 用户名密码或 TOTP 双因子登录 |
| `Dashboard.tsx` | 主框架 | 左侧导航 + 右侧内容，控制所有页面切换 |
| `Sidebar.tsx` | 导航栏 | "Requests / Assets / AI Tools / Settings / Admin" |
| `TeamPage.tsx` | 团队主页 | 需求看板 + 成员列表 |
| `MainPage.tsx` | AI 工具页 | 粘贴代码扫描 + 模板生成 + 自定义生成 |
| `SafetyReport.tsx` | 安全报告 | 展示扫描结果：评分、问题列表、修复建议 |
| `SettingsPage.tsx` | 设置页 | 配 API Key + 个人信息 + 主题 |
| `AdminPage.tsx` | 管理员面板 | 审核待审代码 |
| `ReviewPage.tsx` | 专家审核 | 提交/查看审核请求 |
| `AssetLibrary.tsx` | 资产库 | 查看已保存的代码资产 |
| `ErrorBoundary.tsx` | 错误兜底 | 组件崩溃时显示"Something went wrong"而不是白屏 |

### 什么是 React Context？

Context 是 React 的**全局数据共享**机制。

```tsx
// 不用 Context 时：
<App>
  <Dashboard>
    <Sidebar />
    {/* Sidebar 要用到的数据要从 App → Dashboard → Sidebar 一层层传 */}
  </Dashboard>
</App>

// 用 Context 后：
// ThemeContext 把主题数据存在"全局空间"
// 任何组件都能直接访问，不用层层传递
function Sidebar() {
  const { themePreset, setThemePreset } = useTheme()  // 直接拿
}
```

你的 3 个 Context：

| Context | 存什么 | 谁用 |
|---------|--------|------|
| `ThemeContext` | 主题色、字体、字号 | SettingsPage、全局样式 |
| `SettingsContext` | API Key、模型列表、模型选择 | Dashboard、SettingsPage |
| `ToastContext` | 弹出通知消息 | SettingsPage |

### 什么是 `useState` / `useEffect`？

**`useState`**——让组件有"记忆"。

```tsx
function Counter() {
  // count 是当前值，setCount 是修改它的函数
  const [count, setCount] = useState(0)
  return <button onClick={() => setCount(count + 1)}>{count}</button>
  // 每次点按钮，count 加 1，页面自动更新
}
```

**`useEffect`**——组件加载或更新时执行副作用（发请求、存数据）。

```tsx
function TeamPage() {
  const [team, setTeam] = useState(null)
  
  // 组件加载时，自动从后端请求团队数据
  useEffect(() => {
    fetch('/api/v1/team/my')
      .then(r => r.json())
      .then(data => setTeam(data))
  }, [])  // 空数组 = 只在第一次加载时执行
}
```

## 数据流——页面加载时发生了什么

```
1. 用户打开 http://trufflekit.com
    │
    ▼
2. index.html 加载 → 里面有个 <script src="/src/main.tsx">
    │
    ▼
3. main.tsx 执行 → 创建 React 应用
    │  <ThemeProvider>      ← 提供主题
    │  <SettingsProvider>   ← 提供设置
    │  <ToastProvider>      ← 提供通知
    │  <App />              ← 真正的页面
    │
    ▼
4. App.tsx 检查有没有 JWT token
    │
    ├── 没有 token → 显示 <Login />
    │
    └── 有 token → 验证 token 是否有效
         │
         ├── 无效 → 清空 token → 显示 <Login />
         │
         └── 有效 → 显示 <Dashboard />
              │
              ▼
5. Dashboard.tsx 渲染：
    ├── <Sidebar />               ← 左侧导航栏
    │
    └── 根据 activeNav 显示不同页面：
         ├── ''（默认）→ <TeamPage />        ← 需求看板
         ├── 'ai-tools' → <MainPage />       ← AI 工具
         ├── 'assets'   → <AssetLibrary />   ← 资产库
         ├── 'reviews'  → <ReviewPage />     ← 审核
         ├── 'settings' → <SettingsPage />   ← 设置
         └── 'admin'    → <AdminPage />      ← 管理
```

## 样式系统——CSS 变量 + Tailwind

### CSS 变量是什么

CSS 变量 = 一次定义，多处使用的"颜色常量"。

```css
/* 定义在 index.css 中 */
:root {
  --color-bg: #0D1117;       /* 背景色 */
  --color-accent: #22D3EE;   /* 强调色（青色） */
  --color-text: #E6EDF3;     /* 文字色 */
}
```

组件中这样使用：
```tsx
<div style={{ backgroundColor: 'var(--color-bg)', color: 'var(--color-text)' }}>
```

好处：改一个变量的值，所有用到的地方自动更新。

### Tailwind 是什么

Tailwind 是一套**原子化 CSS**——你不用写 CSS 类名，直接用预定义的类名组合出样式。

```tsx
/* 不用这样写：*/
<div class="my-card">...</div>
/* 然后在 CSS 文件里：.my-card { padding: 16px; border-radius: 8px; } */

/* 而是直接在 HTML 里写：*/
<div className="p-4 rounded-xl border">
  {/* p-4 = padding: 16px, rounded-xl = border-radius: 12px */}
</div>
```

### 5 套主题怎么工作

```
用户点 "Dracula" 主题
    ↓
ThemeContext.setThemePreset('dracula')
    ↓
applyTheme() 把 CSS 变量改成 Dracula 的颜色
    ↓
document.documentElement.style.setProperty('--color-bg', '#282A36')
所有使用 var(--color-bg) 的地方瞬间变色
```

没有重新加载页面，不用切换 CSS 文件——只是改了几个变量的值。

## 学习要点总结

### 你从这个前端项目学到了什么

1. **组件化思维**——一个页面拆成多个独立组件，每个组件只做一件事
2. **单向数据流**——数据从父组件流向子组件，子组件通过回调通知父组件
3. **SPA 原理**——一次加载，JS 负责后续所有页面渲染
4. **CSS 变量 + Tailwind**——不用写 CSS 文件也能做出好看的界面
5. **React Hooks**——useState 存数据，useEffect 发请求
6. **Context 通信**——全局数据不用层层传递
