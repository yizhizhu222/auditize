# 后端——FastAPI + Python

## 技术栈——用了什么、为什么

| 技术 | 用途 | 为什么选它 |
|------|------|-----------|
| **FastAPI** | Web 框架 | Python 最快的 Web 框架，自动生成 API 文档，支持异步 |
| **Uvicorn** | ASGI 服务器 | FastAPI 的推荐服务器，性能高 |
| **SQLite** | 数据库 | 无需安装、无需配置、一个文件搞定 |
| **httpx** | HTTP 客户端 | 异步请求 AI 服务（OpenAI 等） |
| **PyJWT** | JWT 令牌 | 用户登录状态保持 |
| **pyotp** | TOTP 双因子 | 额外的安全层 |
| **psutil** | 系统监控 | 查看 CPU/内存使用情况 |

### 为什么是 FastAPI 不是 Flask？

| 对比 | Flask | FastAPI |
|------|-------|---------|
| Python 版本 | 任何版本 | 需要 3.6+ |
| 异步支持 | 需额外插件 | 原生异步 |
| 自动 API 文档 | 无 | ✅ 自带 `/docs` 页面 |
| 数据验证 | 手动写 | ✅ Pydantic 自动验证 |

FastAPI 更新、更快、自动文档更方便。

### 为什么是 SQLite 不是 MySQL/PostgreSQL？

因为你的项目是自己用或者小团队用（几个到几十个人）。

| 需求 | SQLite | MySQL |
|------|--------|-------|
| 安装 | 不需要 | 要装服务 |
| 配置 | 不需要 | 要配账号密码 |
| 备份 | 复制一个文件 | 要导出导入 |
| 并发写 | 差 | 好 |
| 什么时候换 MySQL | — | 用户超过 1000 人时 |

小项目先用 SQLite，真的做大了再换。

## 核心概念解释

### API 是什么？

API（Application Programming Interface）= 一个"接口"，前端调用后端的方式。

```
前端（浏览器）                   后端（服务器）
    │                              │
    │── POST /api/v1/auth/login ──→│  前端说"我要登录"
    │   {"username":"admin",       │
    │    "password":"admin123"}    │
    │                              │
    │←── 200 {"access_token":"..."}│  后端验证后返回 token
```

你的项目有约 60 个 API 端点，每个端点做一件事。

### RESTful API 是什么？

REST 是一种 API 设计规范：

| 操作 | 方法 | 路径 | 解释 |
|------|------|------|------|
| 创建 | POST | `/api/v1/team/create` | 新建一个团队 |
| 读取 | GET | `/api/v1/team/my` | 查看我的团队 |
| 修改 | PUT | `/api/v1/settings/profile` | 更新个人资料 |
| 删除 | DELETE | `/api/v1/assets/{id}` | 删除一个资产 |

你的 API 都遵循这个规范。

### 什么是 JWT？

JWT = JSON Web Token，一种让用户"拿着令牌通行"的机制。

```
用户登录成功
    │
    ▼
后端生成一串加密字符串（JWT）：
  eyJhbGciOiJIUzI1NiIsInR5cCI6Ik...
    ↓
用户把 JWT 存在浏览器 localStorage 里
    ↓
以后每次发请求，都在 Header 里带上 JWT：
  Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
    ↓
后端用密钥解密 JWT，验证身份
```

JWT 里存了什么：
```json
{
  "sub": "1",           // 用户 ID
  "username": "admin",  // 用户名
  "role": "admin",       // 角色
  "exp": 1700000000     // 过期时间（24 小时后）
}
```

### 什么是装饰器（@）？

Python 的 `@` 语法是**给函数"包装"一层功能**。

```python
@router.post("/login")
async def login(req: LoginRequest):
    # 这个函数只负责登录逻辑
    ...

# 相当于：
# router.post("/login")(login)
# FastAPI 帮你处理了路由匹配、请求解析等工作
```

### 什么是依赖注入（Depends）？

FastAPI 的 Depends = **自动帮你准备参数**。

```python
@router.get("/team/my")
async def get_my_team(
    user: dict = Depends(get_current_user),  # ← 自动从 JWT 解析出用户信息
):
    # 这里直接用 user，不用手动解析 JWT
    print(user["username"])  # fastapi 帮你做了解析工作
```


为什么需要生命周期（lifespan）？
- 应用启动时需要初始化数据库
- 应用关闭时需要清理资源（虽然现在不需要，但保留了扩展性）

### 认证 auth.py——怎么保证安全

```python
# 密码不是明文存的：
password = "admin123"
salt = random_hex(8)                          # 随机 8 字节
hashed = sha256(salt + password)              # 加密后才存数据库
# 登录时：sha256(盐 + 用户输入的密码) == 库里的哈希值？
# 就算数据库泄露，黑客也拿不到原始密码

# TOTP 双因子认证：
# 用户在手机（Google Authenticator）上绑定一个密钥
# 登录时除了密码还要输 6 位动态码
# 动态码每 30 秒变化一次
# 就算密码泄露，没有手机也登录不了
```

### 数据库 db.py——统一管理连接

SQLite 是**文件型数据库**，一个 `.db` 文件就是整个数据库。

```python
# 为什么用 threading.local()？
# FastAPI 是多线程的
# 每个线程需要自己的数据库连接
# threading.local() 让每个线程有自己的"私有变量"
```

3 个数据库文件：

| 数据库 | 存什么 | 为什么分开 |
|--------|--------|-----------|
| `auth.db` | 用户、会话 | 和业务数据隔离，更安全 |
| `app.db` | 团队、需求、资产、扫描结果、审核 | v2 核心数据 |
| `chat_history.db` | 聊天记录 | 独立扩展 |

### 代码生成 generate.py——让 AI 帮你写代码

```python
# 用户输入："做一个订单管理系统"
# 后端组装 prompt：
messages = [
    {"role": "system", "content": "你是一个代码生成器..."},
    {"role": "user", "content": "用 Python 做一个订单管理系统..."}
]

# 调用 AI（比如 OpenAI）：
content = await _call_ai_provider(
    provider="OpenAI",
    api_key="sk-xxx",
    model="gpt-4o",
    messages=messages,
    max_tokens=4096  # 让 AI 别写太长
)

# AI 返回代码后，自动安全扫描：
result = scan_code("python", content)
report = generate_report(result)
```

整个过程：用户说话 → AI 写代码 → 自动安全检查 → 返回结果。

### 安全扫描 scan.py——找出代码里的危险

```python
# 用户粘贴代码 → 扫描 → 报告
#
# 扫描器能发现：
#   - eval()、exec() → 代码注入风险
#   - os.system() → 命令执行风险
#   - password = "xxx" → 硬编码密码
#   - api_key = "sk-xxx" → API Key 泄露
#   - cursor.execute(f"...") → SQL 注入
#   - fetch("http://...") → 意外的网络请求

# 每条发现都有：
#   - 风险等级（critical/high/medium/low）
#   - 说明（非技术人员能看懂）
#   - 在代码的哪一行
#   - 修改建议
```

## 学习要点总结

### 后端开发的核心逻辑

1. **请求 → 处理 → 响应**——每个 API 端点都是这个模式
2. **认证是基础设施**——用 JWT 管理用户登录状态
3. **数据库存持久数据**——SQLite 对小项目够用
4. **不要相信用户输入**——验证、过滤、加锁
5. **异步提高性能**——await 让服务器在等待时干别的事

### 你从后端学到了什么

1. **FastAPI 的工作方式**——装饰器注册路由，函数处理请求
2. **JWT 认证机制**——token 保存状态，不用服务器存 session
3. **RESTful API 设计**——GET/POST/PUT/DELETE 对应增删改查
4. **数据库设计**——用户、团队、需求、资产各自一张表
5. **第三方 API 调用**——用 httpx 调 OpenAI/DeepSeek 等 AI 服务
6. **错误处理**——try/except 捕获异常，给用户友好的错误信息
