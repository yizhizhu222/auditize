# SEC-016: Missing README

| 字段 | 值 |
|---|---|
| 规则编号 | SEC-016 |
| 分类 | 文档 (Documentation) |
| 严重等级 | ⚪ Info |

## 为什么这是问题

README 文件是项目的门面，是开发者了解项目的第一入口。缺少 README 会使其他开发者（包括未来的自己）难以理解项目的目的、如何安装、如何配置以及如何使用。在开源项目中，缺少良好的 README 会显著降低项目的采用率；在内部项目中，它增加了新成员的上手成本和知识传递的障碍。

一个没有 README 的项目传递了"这个项目不被维护"的信号。在安全审计中，缺乏文档的项目更难被正确评估——审计者无法判断哪些文件是重要的、项目如何配置运行，从而可能漏掉安全风险。

## 修复方法

1. **创建 `README.md` 文件**：放在项目根目录。
2. **包含以下核心内容**：
   - 项目名称和简短描述
   - 安装步骤
   - 快速开始指南
   - 环境变量/配置说明
   - 贡献指南（可选）
   - 许可证信息
3. **使用徽章**：CI 状态、覆盖率、许可证等状态徽章增加可信度。
4. **保持更新**：随着项目发展同步更新 README。

## 示例

```markdown
<!-- ❌ 错误的做法：README 过于简单或不存在 -->
# my-project
My project

<!-- ✅ 正确的做法：包含必要信息的完整 README -->

# My Project

[![CI](https://github.com/user/myproject/actions/workflows/ci.yml/badge.svg)](https://...)
[![License](https://img.shields.io/badge/license-MIT-blue.svg)](LICENSE)

A brief description of what this project does and who it's for.

## Prerequisites
- Node.js 20+
- PostgreSQL 15+

## Installation

git clone https://github.com/user/myproject.git
cd myproject
npm install

## Configuration

Create a `.env` file in the project root:

DATABASE_URL=postgresql://localhost:5432/myapp
API_KEY=your_api_key

## Quick Start

npm run dev

Visit http://localhost:3000

## Project Structure

src/
  api/        # API routes
  models/     # Database models
  services/   # Business logic
  utils/      # Utility functions

## Contributing

See CONTRIBUTING.md for details.

## License

MIT
```
