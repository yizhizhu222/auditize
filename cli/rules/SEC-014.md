# SEC-014: Unpinned dependencies

| 字段 | 值 |
|---|---|
| 规则编号 | SEC-014 |
| 分类 | 安全 (Security) |
| 严重等级 | 🟠 High |

## 为什么这是问题

在项目中未锁定依赖版本（如使用 `*`、`>=` 或 `^1.0` 等范围约束）会导致不可复现的构建。依赖项的小版本升级可能引入破坏性的 API 变更，也可能在不知不觉中引入新的安全漏洞。更严重的是，攻击者可能利用依赖混淆攻击或恶意包替换来破坏你的供应链。

当第三方依赖被发现有安全漏洞时，如果不锁定版本，你的项目可能在下次安装时自动升级到受影响的版本。反过来，如果某个依赖引入了破坏性变更（semver breakage），你的 CI/CD 流水线可能在周末的部署中毫无预警地崩溃。

## 修复方法

1. **锁定所有依赖版本**：使用精确版本号（如 `1.2.3` 而不是 `^1.2.3`）。
2. **使用 lock 文件**：确保 `package-lock.json`、`yarn.lock`、`poetry.lock`、`requirements.txt`（固定版本）等被提交到仓库。
3. **配置 Dependabot 或 Renovate**：自动化依赖更新，带有变更日志和 PR 审查。
4. **使用依赖审计工具**：定期运行 `npm audit`、`pip audit`、`yarn audit` 检查已知漏洞。
5. **CI/CD 中检查 lock 文件**：确保 lock 文件与声明式依赖文件一致。

## 示例

```javascript
// ❌ 错误的做法：package.json 使用范围约束
{
  "dependencies": {
    "express": "^4.0.0",
    "axios": "*",
    "lodash": ">=4.0.0"
  }
}
```

```javascript
// ✅ 正确的做法：锁定精确版本
{
  "dependencies": {
    "express": "4.18.2",
    "axios": "1.6.2",
    "lodash": "4.17.21"
  }
}
```

```python
# ❌ 错误的做法：requirements.txt 使用范围
flask>=2.0
requests>=2.25
numpy

# ✅ 正确的做法：锁定精确版本
flask==3.0.0
requests==2.31.0
numpy==1.26.2
```

```dockerfile
# ❌ 错误的做法：不指定 pip 包的版本
RUN pip install flask requests

# ✅ 正确的做法：指定精确版本
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
```
