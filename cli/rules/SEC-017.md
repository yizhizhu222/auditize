# SEC-017: Missing LICENSE

| 字段 | 值 |
|---|---|
| 规则编号 | SEC-017 |
| 分类 | 合规 (Compliance) |
| 严重等级 | ⚪ Info |

## 为什么这是问题

缺少 LICENSE 文件意味着项目的法律状态不明确。在没有明确许可证的情况下，默认适用版权法——这意味着没有人（包括你的用户、贡献者和企业客户）有合法的权利使用、复制、修改或分发你的代码。这会导致法律风险和组织采用障碍。

对于开源项目，缺少许可证会严重限制社区参与。公司通常有政策禁止使用没有明确许可证的开源代码。对于内部项目，缺少许可证也可能在跨团队使用或未来开源时造成法律问题。此外，使用未经适当许可的第三方库可能使用户面临法律纠纷。

## 修复方法

1. **选择一个许可证**：根据项目需求选择合适的开源许可证。
2. **创建 LICENSE 文件**：将完整许可证文本放入项目根目录的 `LICENSE` 文件中。
3. **在 README 中引用**：添加许可证徽章和简短说明。
4. **更新 `package.json`**（Node.js）或 `setup.cfg`（Python）等文件中的许可证字段。
5. **对贡献者明确许可条款**：确保贡献者理解他们的贡献也将以相同的许可证发布。
