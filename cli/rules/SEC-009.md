# SEC-009: Hardcoded absolute paths

| 字段 | 值 |
|---|---|
| 规则编号 | SEC-009 |
| 分类 | 代码质量 (Code Quality) |
| 严重等级 | 🟡 Medium |

## 为什么这是问题

代码中硬编码的绝对路径（如 `/home/user/project/data`、`C:\Users\Admin\app\config`）会严重降低项目的可移植性和可维护性。在不同的环境（开发、测试、生产）或不同的开发人员机器上，这些路径几乎总是不同的。当其他开发者克隆项目或部署到新环境时，硬编码路径会导致程序崩溃或需要手动修改代码。

此外，硬编码路径可能无意中泄露开发者的用户名、内部目录结构和系统配置信息。在 Docker 容器化的部署中，硬编码的主机路径完全不适用，会导致容器部署失败。

## 修复方法

1. **使用环境变量**：将文件路径配置为环境变量或使用 `.env` 文件管理。
2. **使用相对路径**：基于项目根目录使用相对路径。
3. **使用路径构建函数**：使用 `path.join()`、`os.path.join()` 或 `Pathlib` 构建路径。
4. **使用配置系统**：将路径配置放在单独的配置文件中。
5. **在 Linux/macOS 中使用 `$HOME` 和 `~`**：代替用户目录的硬编码路径。

## 示例

```python
import os
from pathlib import Path

# ❌ 错误的做法
DATA_DIR = "/home/zhang/project/data"
LOG_PATH = "/var/log/myapp/app.log"
CONFIG_PATH = "C:\\Users\\Admin\\app\\config.ini"

def load_data():
    with open("/home/zhang/project/data/dataset.csv") as f:
        return f.read()
```

```python
# ✅ 正确的做法
from pathlib import Path
import os

# 基于项目根目录
BASE_DIR = Path(__file__).resolve().parent.parent
DATA_DIR = BASE_DIR / "data"
LOG_DIR = os.getenv("LOG_DIR", str(BASE_DIR / "logs"))
CONFIG_PATH = os.getenv("CONFIG_PATH", str(BASE_DIR / "config" / "config.ini"))

def load_data():
    with open(DATA_DIR / "dataset.csv") as f:
        return f.read()

# 或者使用配置对象
class AppConfig:
    def __init__(self):
        self.base_dir = Path(__file__).resolve().parent.parent
        self.data_dir = Path(os.getenv("DATA_DIR", str(self.base_dir / "data")))
        self.log_path = Path(os.getenv("LOG_DIR", str(self.base_dir / "logs")))
```
