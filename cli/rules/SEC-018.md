# SEC-018: Missing CI/CD

| 字段 | 值 |
|---|---|
| 规则编号 | SEC-018 |
| 分类 | 运维 (Operations) |
| 严重等级 | 🟡 Medium |

## 为什么这是问题

缺少 CI/CD 流水线意味着每次代码变更都必须手动构建、测试和部署。这不仅低效且容易出错，还带来严重的安全风险。没有自动化测试验证，有缺陷的代码可能被直接合并到主分支并部署到生产环境；没有自动化安全检查，依赖漏洞、硬编码密钥、安全配置错误等问题只能在事后被发现。

手动部署流程缺乏审计追踪——无法追溯谁在什么时候部署了什么版本。在发生安全事件时，这种问责缺失会严重阻碍调查和恢复。此外，没有 CI/CD 的项目通常缺乏标准化的代码质量门禁，导致代码质量逐渐退化。

## 修复方法

1. **选择合适的 CI/CD 平台**：GitHub Actions、GitLab CI、Jenkins、CircleCI 等。
2. **配置基本流水线**：至少包含代码检出、安装依赖、运行测试。
3. **添加安全检查步骤**：
   - 依赖扫描（Dependabot、Snyk）
   - 代码静态分析（Linter、SAST）
   - 密钥检测（如 Auditize）
   - Docker 镜像扫描
4. **配置分支保护规则**：要求 CI 通过后才能合并 PR。
5. **添加自动化部署**：至少自动化到测试/暂存环境。

## 示例

```yaml
# ❌ 错误的做法：没有 CI/CD 配置文件
# 项目中没有 .github/workflows/ 或 .gitlab-ci.yml

# ✅ 正确的做法：GitHub Actions 配置示例
# .github/workflows/ci.yml
name: CI/CD Pipeline

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Lint code
        run: npm run lint

  test:
    needs: lint
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install dependencies
        run: npm ci
      - name: Run tests
        run: npm test
      - name: Run security audit
        run: npm audit

  security:
    needs: lint
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Secret scanning
        uses: trufflesecurity/trufflehog@v3
        with:
          path: ./
```
