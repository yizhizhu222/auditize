# SEC-020: Default admin password

| 字段 | 值 |
|---|---|
| 规则编号 | SEC-020 |
| 分类 | 安全 (Security) |
| 严重等级 | 🔴 Critical |

## 为什么这是问题

在代码中使用默认的管理员密码（如 `admin/admin`、`admin/123456`、`root/root`）是极为常见但危害巨大的安全漏洞。攻击者会首先尝试这些众所周知的默认凭证来尝试登录系统。OWASP 和多个漏洞数据库均将默认凭证列为最常被利用的攻击向量之一。

即使是用于开发环境的默认密码，一旦代码被提交到仓库，这些凭证就会暴露给所有可以访问仓库的人。更危险的是，开发者在部署到生产环境后经常忘记更改默认密码。自动化扫描工具和攻击者会系统地尝试默认密码组合，扫描成功率极高。

## 修复方法

1. **从代码中移除所有默认密码**：不要在任何代码文件中硬编码默认凭据。
2. **强制首次登录修改密码**：系统设计应强制用户在首次登录时修改默认密码。
3. **使用环境变量或密钥管理服务**：初始密码通过安全渠道传递。
4. **实施强密码策略**：要求密码长度 >= 12、包含大小写字母和特殊字符。
5. **添加配置验证**：启动时检查是否仍在使用默认凭据并拒绝启动。

## 示例

```python
# ❌ 错误的做法
class AdminConfig:
    DEFAULT_USERNAME = "admin"
    DEFAULT_PASSWORD = "admin123"  # 硬编码的默认密码

def create_admin_user():
    user = User(
        username=AdminConfig.DEFAULT_USERNAME,
        password=AdminConfig.DEFAULT_PASSWORD,
        role="admin"
    )
    db.session.add(user)
```

```python
# ✅ 正确的做法
import os
import secrets
from werkzeug.security import generate_password_hash

def create_admin_user():
    username = os.getenv("ADMIN_USERNAME", "admin")
    # 首次启动时生成随机密码
    initial_password = os.getenv("INITIAL_ADMIN_PASSWORD")

    if not initial_password:
        initial_password = secrets.token_urlsafe(24)
        print(f"!!! Initial admin password: {initial_password}")
        print("!!! Change this password immediately after first login")
        # 实际项目中应该通过安全渠道发送密码

    # 强制首次登录修改密码
    user = User(
        username=username,
        password_hash=generate_password_hash(initial_password),
        role="admin",
        must_change_password=True  # 强制修改密码标记
    )
    db.session.add(user)
```

```javascript
// ❌ 错误的做法
const defaultAdmin = {
    email: "admin@example.com",
    password: "Password123!"
};

// ✅ 正确的做法
const password = process.env.INITIAL_ADMIN_PASSWORD;
if (!password || password === "Password123!") {
    throw new Error("Must set a strong initial admin password");
}
```
