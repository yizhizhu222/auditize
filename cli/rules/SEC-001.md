# SEC-001: Hardcoded API Keys/Secrets

| 字段 | 值 |
|---|---|
| 规则编号 | SEC-001 |
| 分类 | 安全 (Security) |
| 严重等级 | 🔴 Critical |

## 为什么这是问题

在代码仓库中硬编码 API 密钥、数据库密码、JWT 密钥或云服务凭证是最常见且最严重的安全漏洞之一。一旦代码被推送至远程仓库（尤其是公开仓库），凭证会立即暴露给所有访问者。攻击者可以利用这些凭证未经授权访问你的云服务、数据库或第三方 API，导致数据泄露、财务损失和合规违规。

即使后续删除了凭证，它们仍然会保留在 Git 历史中，成为长期的安全隐患。自动化爬虫会持续扫描 GitHub 等平台以发现泄露的凭证。

## 修复方法

1. **立即撤销泄露的凭证**：在对应的服务提供商处轮换密钥/密码。
2. **将所有凭证移至环境变量**：使用 `.env` 文件（确保已加入 `.gitignore`）存储敏感值。
3. **使用密钥管理服务**：对于生产环境，使用 Vault、AWS Secrets Manager 或类似服务。
4. **在代码中使用环境变量读取**：如 `os.getenv("API_KEY")` 或 `process.env.API_KEY`。
5. **清理 Git 历史**：使用 `git filter-branch` 或 `BFG Repo-Cleaner` 从历史中移除凭证。

## 示例

```python
# ❌ 错误的做法
API_KEY = "sk-abc123def456ghi789jkl012"
DB_PASSWORD = "P@ssw0rd2024!"

def call_api():
    return requests.post("https://api.example.com/data",
                         headers={"Authorization": f"Bearer {API_KEY}"})
```

```python
# ✅ 正确的做法
import os
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("API_KEY")
DB_PASSWORD = os.getenv("DB_PASSWORD")

if not API_KEY:
    raise ValueError("API_KEY environment variable not set")

def call_api():
    return requests.post("https://api.example.com/data",
                         headers={"Authorization": f"Bearer {API_KEY}"})
```
