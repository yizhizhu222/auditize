# SEC-006: Debug statements in production code

| 字段 | 值 |
|---|---|
| 规则编号 | SEC-006 |
| 分类 | 代码质量 (Code Quality) |
| 严重等级 | 🟠 High |

## 为什么这是问题

在代码中遗留的调试语句（如 `console.log()`、`print()`、`var_dump()`、`debugger`、`dd()` 等）可能导致敏感信息泄露，包括数据库查询、用户数据、会话令牌或内部系统架构细节。在生产环境中，这些输出可能被用户、日志聚合系统或攻击者看到，成为攻击者收集情报的入口。

此外，无谓的日志输出会影响性能、增加日志存储成本，并掩盖真正重要的生产日志信息。包含 `debugger` 语句的 JavaScript 代码会导致浏览器执行到该处时暂停执行，影响用户体验。

## 修复方法

1. **全局搜索常见的调试关键字**：搜索 `console.log`、`print`、`debugger`、`var_dump`、`println` 等。
2. **使用日志库**：使用专业的日志库（如 Python 的 `logging`、Node.js 的 `winston`），设置不同级别（DEBUG、INFO、ERROR）。
3. **配置生产环境的日志级别**：生产环境只显示 WARNING/ERROR 级别，DEBUG 级别仅在开发环境启用。
4. **添加代码审查规则**：配置 Linter 规则（如 ESLint `no-console`、`no-debugger`）在 CI/CD 中阻止包含调试语句的代码合并。
5. **使用断点而不是打印语句**：开发时使用 IDE 的调试器功能而不是打印语句。

## 示例

```javascript
// ❌ 错误的做法
function processPayment(userId, amount) {
    console.log("Processing payment for user:", userId);
    console.log("Amount:", amount);
    // 可能还包含了调试用的敏感信息
    console.log("Session token:", session.token);
    debugger;  // 生产代码中的断点
    // ... 处理逻辑
    console.log("SQL query:", sqlQuery);
}
```

```javascript
// ✅ 正确的做法
const logger = require("winston");

logger.configure({
    level: process.env.NODE_ENV === "production" ? "warn" : "debug",
});

function processPayment(userId, amount) {
    logger.debug("Processing payment", { userId: maskUserId(userId), amount });
    // ... 处理逻辑
    logger.info("Payment processed successfully", { paymentId });
}
```
