# SEC-004: .gitignore incomplete

| 字段 | 值 |
|---|---|
| 规则编号 | SEC-004 |
| 分类 | 安全 (Security) |
| 严重等级 | 🟡 Medium |

## 为什么这是问题

不完整的 `.gitignore` 文件可能导致敏感或不需要的文件被意外提交到仓库中。常见的遗漏包括操作系统文件（`.DS_Store`、`Thumbs.db`）、IDE 配置（`.idea/`、`.vscode/`）、构建产物（`dist/`、`build/`）、日志文件（`*.log`）和依赖目录（`node_modules/`、`vendor/`）。

这些文件中有些可能包含敏感信息（如 IDE 的本地配置文件中可能包含数据库密码或 API 端点），有些会增加仓库体积和克隆时间（如 `node_modules`），还有些会导致跨平台协作问题。

## 修复方法

1. **审查当前 `.gitignore` 的内容**：检查是否涵盖了项目类型相关的最佳实践模式。
2. **使用 `gitignore.io` 生成模板**：根据项目使用的语言、框架和 IDE 生成全面的 `.gitignore`。
3. **检查已跟踪的不需要的文件**：运行 `git ls-files` 检查是否有本应被忽略的文件已被跟踪。
4. **移除已跟踪的不需要的文件**：使用 `git rm --cached <file>` 停止跟踪但不删除本地文件。
5. **添加对以下类别的全面覆盖**：操作系统文件、IDE 文件、依赖目录、构建产物、日志文件、环境配置文件、密钥文件。

## 示例

```
# ❌ 错误的做法：.gitignore 过于精简
node_modules/

# ✅ 正确的做法：覆盖多类文件
# 依赖
node_modules/
vendor/
.pnp.js

# 环境变量
.env
.env.*
!.env.example

# 操作系统
.DS_Store
Thumbs.db
*.swp

# IDE
.idea/
.vscode/
*.sublime-project
*.sublime-workspace

# 构建输出
dist/
build/
*.tsbuildinfo

# 日志
*.log
npm-debug.log*
yarn-debug.log*
yarn-error.log*

# 密钥
*.pem
*.key
*.cert
```
